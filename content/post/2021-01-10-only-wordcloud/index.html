おはようございます！ 今日は、テキストデータを分析して、ワードクラウドなるものを作ろうと思います！

本当は、Twittterからテキストデータを取得して、ワードクラウドを作るところまでを記事にしたかったのですが、 パッケージrtweetは、TwitterのAPIなしでは、RMarkdownで使えませんでした！ なので、テキストデータの取得＆データを整えるところは、また今度の機会にnoteにしようと思います！

では、コードを書いていきましょう！

とにかく全コードをみたい人は、一番下の全コードを参照してください。

環境

Rstudio

R version 4.0.3 (2020-10-10)　

Windows10　

パッケージを用意する

Packages <- c("tidyverse","RMeCab", "wordcloud", "wordcloud2")
lapply(Packages, library, character.only = TRUE)

データを読み込む

今回は、事前にツイッターからテキストデータをダウンロードしました。 鬼滅の刃が人気なので、鬼滅 を含むツイートを1000件取得し、事前にデータ解析しやすい形に整えておきました。

形態素解析を実行する

ここからは、RMeCabでの作業になります。 今回は、** 鬼滅 **に対する印象を調べたかった為、形容詞と名詞を抽出しました。 サ変接続は、動詞として扱うため除きました。非自立の形容詞は、品詞としてトピックの印象をよく表さない為、排除しています。

docDF_text = docDF("tweets_text_all.txt", type = 1, pos = c("名詞", "形容詞")) %>% 
  select(everything(), 
         word = TERM,
         freq = starts_with("tweets"))


docDF_text2 = docDF_text %>% 
  filter(!POS2 %in% c("非自立", "サ変接続"))

wordcloudを作る

まずは、パッケージwordcloudを使ってグラフを作ります。

wordcloud(docDF_text2$word, docDF_text2$freq, min.freq=20, max.words = Inf, 
          scale = c(3.5, 0.25), # c(size, width)
          ordered.colors = FALSE,
          random.order = FALSE, random.color = FALSE, rot.per = 3.5, 
          colors = brewer.pal(10, "Set3")) #brewer.pal(8, "Dark2"))

さすがにしょぼすぎるので、パッケージWordcloud2を使ってワードクラウドを作りましょう。

df <- docDF_text2 %>% 
  arrange(desc(freq)) %>% 
  filter(freq < 1000) %>% 
  select(word, freq)


wordcloud2(data = df, size = 1, 
           minRotation = -pi/6, maxRotation = -pi/6,
           color = ifelse(df[,2] > 100, "red", "skyblue"),
           shuffle = FALSE, shape = 'pentagon')

鬼滅商品の売れ行きがえぐいORツイッターにて鬼滅商戦が発生している模様。
